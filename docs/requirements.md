## 1. 项目概述

### 1.1 项目背景

高校软件小组数字化平台是为高校软件小组团队定制的一体化管理与服务平台，旨在实现团队运营数字化、信息沟通实时化、成员成长智能化。在现有Web站基建成果上，确立"**优先APP化**"的核心战略，快速拓展移动端专属服务，并逐步向业务闭环、智能化与数据生态演进。

### 1.2 建设目标

| 阶段 | 核心目标 | 时间跨度 |
|------|----------|----------|
| 第一阶段 | Web端基础管理平台搭建，实现团队运营数字化 | 已完成 |
| 第二阶段 | 全面APP化与移动服务拓展，提升成员活跃度 | 近期（当前重点） |
| 第三阶段 | 业务深度闭环与管理精细化，实现数据驱动运营 | 中期 |
| 第四阶段 | AI赋能与生态化，构建智能化主动服务体系 | 远期 |

### 1.3 核心价值

- **管理提效**：标准化团队日常管理流程，减少人工操作
- **信息触达**：多端消息推送，确保重要通知实时送达
- **数据沉淀**：完整记录团队成长轨迹，积累数字资产
- **智能赋能**：AI辅助决策与个性化服务，助力成员成长

## 2. 技术栈约束

> ⚠️ 以下内容属于**不可协商约束**，所有开发必须严格遵守

### 2.1 服务端技术栈

| 层级 | 技术选型 | 版本要求 |
|------|----------|----------|
| 编程语言 | Java | JDK 1.8+ |
| Web框架 | JSP + Servlet | Servlet 4.0 (javax.*) |
| 数据访问 | 原生 JDBC | - |
| 数据库 | MySQL | 8.0+ |
| Web容器 | Apache Tomcat | 9.x |
| 构建工具 | Maven | 3.6+ |
| 会话管理 | HttpSession | - |
| 权限控制 | Servlet Filter | - |
| 前端框架 | Bootstrap 5 + Tabler Admin Template | - |
| 文件存储 | 本地文件系统（外部目录） | - |
| AI能力 | 多LLM Provider支持（MiniMax/豆包/文心/通义/OpenAI） | - |

### 2.2 移动端技术选型（第二阶段）

| 方案 | 说明 | 优先级 |
|------|------|--------|
| 微信小程序 | 跨平台、无需安装、用户门槛低 | 推荐（首选） |
| Flutter | 跨平台原生体验，一套代码多端运行 | 备选 |
| React Native | 生态成熟，前端团队友好 | 备选 |

### 2.3 明确禁止

* ❌ Spring / Spring Boot / Spring MVC
* ❌ MyBatis / Hibernate / JPA
* ❌ Lombok
* ❌ NoSQL / Redis（当前阶段）
* ❌ 服务端引入微服务框架（保持单体架构，预留MCP接口化能力）

## 3. 用户角色定义（RBAC）

### 3.1 角色体系

| 角色 | 代码 | 说明 | 核心权限 |
|------|------|------|----------|
| 游客 | GUEST | 未登录用户 | 浏览公开信息、招新报名 |
| 小组成员 | MEMBER | 正式在组学生 | 个人信息维护、活动/项目参与、奖项提交 |
| 项目管理员 | PROJECT_ADMIN | 项目负责人 | 项目管理、成员审批、计划制定 |
| 系统管理员 | ADMIN | 团队管理层 | 全局管理、审批、数据统计 |
| 教师 | TEACHER | 指导老师（预留） | 指导查看、高级审批（待开放） |

### 3.2 各角色核心功能

#### 游客（未登录）
- 浏览小组新闻、活动、项目公开信息
- 浏览小组介绍与历史荣誉
- 填写招新报名表
- 使用AI助手进行基础咨询

#### 软件小组成员（MEMBER）
- 登录系统
- 维护个人基础信息与头像
- 上传获奖证书/图片
- 查看与报名小组活动
- 申请项目并组队，参与项目管理
- 查看个人奖项、活动、项目记录
- 群组聊天、消息通知
- 在线学习、知识评测
- 考勤签到打卡
- 使用AI助手协助完成操作

#### 系统管理员（ADMIN）
- 发布/编辑/下线新闻与通知
- 审批项目申请、奖项申请、招新申请
- 管理活动、奖项、成员、项目
- 查看用户登录情况与操作日志
- 进行全局数据统计与看板查看
- 群组管理、系统配置
- 使用AI助手进行管理操作

#### 教师（TEACHER）
- **预留角色，当前版本不开放任何功能**

## 10. 核心业务流程

### 10.1 招新报名流程（游客 → 成员）

```
游客填写报名表 → 提交（status=1待处理） → 管理员审核
     ├─ 同意：自动创建user + member_profile，status=2
     └─ 拒绝：status=0
```

### 10.2 奖项提交流程（成员 → 审批 → 归档）

```
成员提交奖项 → 上传证书图片 → 管理员审核
     ├─ 通过：award_status=APPROVED，公开展示
     └─ 拒绝：award_status=REJECTED
```

### 10.3 项目管理流程（申请 → 审批 → 执行 → 结项）

```
成员申请创建项目 → 自动成为项目管理员 → 管理员审批
     ├─ 通过：status=APPROVED → 成员申请加入 → 项目管理员审批
     │                                          ├─ 通过：加入项目
     │                                          └─ 拒绝：驳回
     │                    → 制定计划 → 更新进度 → 完成结项
     └─ 拒绝：status=REJECTED
```

项目状态流转：`PENDING → APPROVED → IN_PROGRESS → COMPLETED`

### 10.4 活动报名流程（时间窗口控制）

```
活动创建（设置报名起止时间、名额） → 成员在窗口期内报名（status=PENDING）
     → 管理员审核（报名截止前）
          ├─ 通过：status=CONFIRMED
          ├─ 拒绝：status=REJECTED
          └─ 过期未审：自动标记为EXPIRED
```

### 10.5 新闻发布流程

```
管理员创建新闻 → 富文本编辑正文 → 正文保存为HTML文件 → 数据库记录路径
     → 发布（status=1） → 前端公开展示
     → 下线（status=0）
```

## 14. 开发注意事项

1. **密码加密**：使用DES加密，密钥配置在 `config.Config.getDesKey()`，**禁止修改加密算法**（会导致历史密码失效）

2. **文件上传**：统一通过 `FileStorageServlet` 和 `FileUtil` 处理，文件存储到外部 `localstorage/` 目录，数据库只存file_id和逻辑路径

3. **会话管理**：使用HttpSession，默认超时30分钟，移动端同时支持Token认证

4. **状态管理**：使用明确的字符串状态值（如PENDING/APPROVED），避免使用魔法数字，状态流转要记录历史

5. **Servlet路由规范**：
   - 所有Servlet支持action参数路由
   - GET请求处理查询和页面跳转（通过`<a href>`触发的操作）
   - POST请求处理表单提交和数据修改
   - 建议所有action同时在doGet和doPost中实现，保证兼容性

6. **URL编码**：所有重定向URL中的中文参数必须使用 `URLEncoder.encode()` 编码

7. **XSS防护**：用户输入经过SecurityFilter的Jsoup清洗，JSP页面输出使用JSTL `<c:out>` 或 `${fn:escapeXml()}`

8. **软删除**：删除操作使用 `deleted=1` 标记，不做物理删除，查询时默认过滤已删除数据

9. **配置管理**：敏感配置（数据库密码、AI Key）写入 `config.local.properties`，**严禁提交到Git**

10. **Servlet/Filter注册**：注册位置分散在web.xml和@WebServlet注解，修改时务必两处都检查

11. **FileUtil使用**：新代码必须使用 `FileUtil.getCategoryDir()` 和 `FileUtil.resolvePhysicalPath()`，不要用 `getServletContext().getRealPath()`

12. **Maven构建**：使用 `mvn clean package` 打包，没有Tomcat插件，需部署到外部Tomcat 9
