## 四、本地文件与 HTML 存储规范（强制）

### 4.1 根目录结构

```
webapps/Software_group/
├── localstorage/
│   ├── news/              # 新闻HTML文件
│   │   ├── award/
│   │   ├── activity/
│   │   └── notice/
│   ├── files/             # 用户上传文件
│   │   └── {userId}/
│   └── images/            # 图片资源
├── static/
│   ├── jsp/               # 公共JSP
│   ├── admin/             # 管理员页面
│   └── member/            # 成员页面
└── WEB-INF/
    └── web.xml
```

### 4.2 新闻 / 通知存储规则

* 新闻正文保存为 **HTML 文件**
* 存储目录：`localstorage/news/{type}/{id}.html`
* 数据库 **只保存相对路径**

### 4.3 文件 / 图片存储规则

* 存储目录：`localstorage/files/{userId}/`
* 本地文件名：`{timestamp}_{uuid}.{ext}`
* 通过 `file_storage` 表统一管理元数据

## 十五、开发注意事项

1. **密码加密**：使用 DES 加密，密钥配置在 `config.Config.DES_KEY`
2. **文件上传**：通过 `FileStorageServlet` 统一管理，返回 file_id
3. **会话管理**：使用 HttpSession，30分钟超时
4. **状态管理**：使用 status 字段（1=启用，0=禁用）
5. **Servlet Action 处理规范**：
   - 所有需要通过链接（`<a href>`）触发的操作（如 approve、reject、delete 等）必须在 `doGet()` 方法的 switch 语句中处理
   - 所有通过表单 POST 触发的操作在 `doPost()` 方法中处理
   - JSP 页面中的按钮如果使用 `<a href>` 链接，则对应 action 必须在 doGet 中实现
   - JSP 页面中的按钮如果使用 `<form method="POST">`，则对应 action 可只在 doPost 中实现
   - 为保证一致性，**建议所有 action 都同时在 doGet 和 doPost 中实现**
6. **URL重定向编码**：所有重定向URL中的中文消息必须使用URLEncoder.encode()进行编码，避免Unicode编码错误

**报名状态计算逻辑：**
```java
// 动态计算报名状态
String calculateStatus(Registration reg, Activity activity) {
    if (reg.status.equals("confirmed")) {
        return "申请已确认";
    } else if (reg.status.equals("rejected")) {
        return "申请已驳回";
    } else if (reg.status.equals("pending")) {
        // 检查是否超过报名截止时间
        if (currentTime > activity.registration_end_time) {
            // 自动更新为过期状态
            reg.status = "expired";
            return "申请已过期";
        } else {
            return "申请审核中";
        }
    }
}
```

**状态说明：**
- **申请已确认**：管理员已通过审核
- **申请审核中**：已报名，等待管理员确认，且在报名截止时间内
- **申请已驳回**：管理员驳回了报名申请
- **申请已过期**：报名截止时间已过但管理员未审核（自动计算）

**管理员权限逻辑：**
- **未过期活动**（当前时间 <= registration_end_time）：
  - 可查看报名列表
  - **待审核(pending)**：显示【通过】和【驳回】按钮

  - **已确认(confirmed)**：不显示任何操作按钮（报名已通过，不可再拒绝）
  - 支持全选/多选后批量通过
  - 可编辑活动信息

- **已过期活动**（当前时间 > registration_end_time）：
  - 可查看报名列表（只读）
  - **不可进行任何操作**
  - 显示状态标签（已确认/已过期等）

#### 15.4.5 活动删除规则

活动删除需要满足以下条件：

| 条件 | 说明 |
|------|------|
| ✅ 可删除 | 活动未开始（upcoming）且无已确认报名 |
| ❌ 不可删除 | 已有学生确认参加（confirmed） |
| ❌ 不可删除 | 活动状态为已结束（completed）或进行中（ongoing） |

**删除时执行的操作：**
1. 检查是否有已确认的报名 → 有则拒绝删除
2. 检查活动状态 → 已结束/进行中则拒绝删除
3. 删除该活动的所有报名记录（pending、rejected等全部删除）
4. 删除活动本身

**技术实现：**
- 删除按钮使用 GET 请求（`<a href="...action=delete&id=...">`）
- delete action 在 ActivityServlet 的 doGet 和 doPost 中都实现
- 删除前调用 `registrationDAO.getConfirmedCount()` 检查已确认数量
- 删除时调用 `registrationDAO.deleteByActivityId()` 删除所有报名记录

活动分类使用字典表(ACTIVITY_TYPE)，在创建/编辑活动时从下拉菜单选择：

| 字典代码 | 名称 |
|----------|------|
| LECTURE | 讲座 |
| SEMINAR | 座谈 |
| TEA_PARTY | 茶话会 |
| PROJECT_INTRO | 项目介绍会 |
| WORKSHOP | 工作坊 |
| COMPETITION | 竞赛 |
| TRAINING | 培训 |
| TEAM_BUILDING | 团建 |
| OTHER | 其他 |
