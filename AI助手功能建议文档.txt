1.1 项目概述
--------------------------------------------------------------------------------
高校软件小组管理系统是一个基于 Maven + JSP + Servlet + JDBC 的 Java Web 应用，
专为高校软件小组设计的管理平台。

1.2 技术架构
--------------------------------------------------------------------------------
- 后端框架：Java 1.8 + Servlet 4.0
- 持久层：JDBC + HikariCP 数据库连接池
- 前端技术：JSP + JSTL + Bootstrap 5 + Tabler Admin
- 数据库：MySQL 8.0+
- Web容器：Apache Tomcat 9.0+
- 构建工具：Maven
- 加密方式：DES加密

1.3 现有功能模块
--------------------------------------------------------------------------------
- 用户管理（登录、登出、权限控制）
- 新闻管理（发布/编辑/删除）
- 招新管理（报名表单、后台审核）
- 成员管理（个人档案、信息维护）
- 活动管理（活动发布、时间窗口报名）
- 奖项管理（提交/审批、图片上传）
- 项目管理（申请/审批、成员管理）
- 日志管理（操作日志自动记录）
- 文件存储（统一上传/下载/管理）

1.4 项目包结构
--------------------------------------------------------------------------------
- config/      - 配置类（Config, ConfigManager）
- dao/         - 数据访问对象（14个DAO类）
- filter/      - Servlet过滤器（AuthFilter, LoggingFilter等）
- model/       - 实体类（18个Model，与数据库表对应）
- servlet/     - Servlet控制器（16个Servlet）
- util/        - 核心工具类（DBUtil, DESUtil, FileUtil, StringUtil）

                           第二部分：AI助手功能建议
================================================================================

2.1 功能定位
--------------------------------------------------------------------------------
在现有管理系统基础上添加AI助手功能，旨在提升用户体验和管理效率。
AI助手可作为系统的智能辅助模块，为不同角色提供个性化服务。

2.2 推荐方案：多功能AI助手平台
--------------------------------------------------------------------------------
方案概述：构建一个统一的AI助手平台，集成智能客服、智能搜索、数据分析等
多种功能。通过Servlet架构接入AI服务，为管理员、成员、游客提供差异化AI服务。

2.2.1 核心功能模块

（一）智能客服问答
    - 解答系统使用问题（如何提交奖项、如何报名活动等）
    - 回答关于小组、成员、活动、奖项的常见问题
    - 提供操作指引和流程说明
    - 基于知识库的FAQ自动回复

（二）智能搜索增强
    - 语义化搜索新闻、活动、奖项内容
    - 自然语言查询（如"最近有什么活动？"）
    - 多条件组合查询支持
    - 搜索结果智能排序和推荐

（三）管理员AI助手
    - 数据统计分析和建议
    - 操作日志智能分析
    - 异常行为预警提醒
    - 自动化报表生成辅助

（四）成员发展助手
    - 个人成就汇总展示
    - 奖项/活动参与建议
    - 项目组队智能匹配
    - 个人简历智能生成辅助

（五）内容生成助手
    - 新闻稿/通知文案生成
    - 奖项申请文案辅助
    - 项目计划书模板生成
    - 活动总结自动生成

3.1 架构设计
--------------------------------------------------------------------------------
基于现有MVC架构，新增AI服务层：
- View层：新增AI助手JSP页面（WebContent/jsp/ai/）
- Controller层：新增AIServlet控制器（servlet/AIServlet.java）
- Service层：新增AIService服务类（service/AIService.java）
- DAO层：扩展现有DAO或新建AI相关表（dao/）
- Model层：新增AI相关实体类（model/AIConversation.java等）

3.2 AI服务集成方案
--------------------------------------------------------------------------------

【方案A】LLM API集成（推荐）
    - 接入百度文心一言/阿里通义千问/OpenAI API
    - 通过HTTPS调用远程AI服务
    - 支持流式响应（Server-Sent Events）
    - 需要API Key配置和调用计费

【方案B】本地轻量模型
    - 部署轻量级开源模型（如ChatGLM、Qwen本地版）
    - 无需外网调用，保护数据隐私
    - 需要服务器有较强GPU/CPU资源
    - 模型更新和维护成本较高

【方案C】规则引擎+知识库（混合方案）
    - 基于规则和知识库的FAQ匹配
    - 简单问题本地处理，复杂问题转发LLM
    - 降低AI服务调用成本
    - 适合FAQ问答场景

    package servlet;

    import service.AIService;
    import model.AIConversation;
    import model.AIMessage;

    import javax.servlet.ServletException;
    import javax.servlet.http.HttpServlet;
    import javax.servlet.http.HttpServletRequest;
    import javax.servlet.http.HttpServletResponse;
    import javax.servlet.http.HttpSession;
    import java.io.IOException;
    import java.io.PrintWriter;
    import java.util.List;

    public class AIServlet extends HttpServlet {

        private AIService aiService = new AIService();

        @Override
        protected void doGet(HttpServletRequest req, HttpServletResponse resp)
                throws ServletException, IOException {
            String action = req.getParameter("action");

            if ("chat".equals(action)) {
                // 显示聊天页面
                req.getRequestDispatcher("/jsp/ai/ai_chat.jsp").forward(req, resp);
            } else if ("history".equals(action)) {
                // 获取聊天历史
                getChatHistory(req, resp);
            } else {
                resp.sendRedirect(req.getContextPath() + "/ai?action=chat");
            }
        }

        @Override
        protected void doPost(HttpServletRequest req, HttpServletResponse resp)
                throws ServletException, IOException {
            req.setCharacterEncoding("UTF-8");
            resp.setCharacterEncoding("UTF-8");
            resp.setContentType("application/json;charset=UTF-8");

            String action = req.getParameter("action");

            if ("send".equals(action)) {
                sendMessage(req, resp);
            } else if ("clear".equals(action)) {
                clearConversation(req, resp);
            }
        }

        private void sendMessage(HttpServletRequest req, HttpServletResponse resp)
                throws IOException {
            String message = req.getParameter("message");
            String sessionId = req.getParameter("session_id");
            String aiResponse = aiService.getAIResponse(message, sessionId);

            PrintWriter out = resp.getWriter();
            out.print("{\"success\":true,\"response\":\"" + aiResponse + "\"}");
            out.flush();
        }

6.2 AIService核心代码结构
--------------------------------------------------------------------------------

    package service;

    import util.AIClientUtil;
    import dao.AIConversationDAO;
    import dao.AIMessageDAO;
    import model.AIMessage;
    import model.AIConversation;

    import java.util.List;

    public class AIService {

        private AIClientUtil aiClient = AIClientUtil.getInstance();
        private AIConversationDAO conversationDAO = new AIConversationDAO();
        private AIMessageDAO messageDAO = new AIMessageDAO();

        public String getAIResponse(String userMessage, String sessionId) {
            // 1. 获取或创建会话
            AIConversation conversation = conversationDAO.findBySessionId(sessionId);
            if (conversation == null) {
                conversation = new AIConversation();
                conversation.setSessionId(sessionId);
                conversationDAO.save(conversation);
            }

            // 2. 保存用户消息
            AIMessage userMsg = new AIMessage();
            userMsg.setConversationId(conversation.getId());
            userMsg.setRole("user");
            userMsg.setContent(userMessage);
            messageDAO.save(userMsg);

            // 3. 构建上下文（包含知识库检索结果）
            String context = buildContext(userMessage);

            // 4. 调用AI服务
            String aiResponse = aiClient.chat(context, userMessage);

            // 5. 保存AI回复
            AIMessage assistantMsg = new AIMessage();
            assistantMsg.setConversationId(conversation.getId());
            assistantMsg.setRole("assistant");
            assistantMsg.setContent(aiResponse);
            messageDAO.save(assistantMsg);

            return aiResponse;
        }

        private String buildContext(String userMessage) {
            // 从知识库检索相关内容
            // 构建系统提示词
            return "你是高校软件小组管理系统的AI助手...";
        }

        public List<AIMessage> getConversationHistory(String sessionId) {
            AIConversation conversation = conversationDAO.findBySessionId(sessionId);
            if (conversation != null) {
                return messageDAO.findByConversationId(conversation.getId());
            }
            return null;
        }
    }

7.1 AI助手对话页面（ai_chat.jsp）
--------------------------------------------------------------------------------

    <%@ page contentType="text/html;charset=UTF-8" language="java" %>
    <%@ taglib prefix="c" uri="http://java.sun.com/jsp/jstl/core" %>
    <!DOCTYPE html>
    <html>
    <head>
        <title>AI助手 - 高校软件小组管理系统</title>
        <link rel="stylesheet" href="${pageContext.request.contextPath}/dist/css/tabler.min.css">
    </head>
    <body>
        <div class="page-wrapper">
            <div class="page-header">
                <h1>AI智能助手</h1>
            </div>
            <div class="chat-container" id="chatContainer">
                <div class="messages" id="messages">
                    <div class="message assistant">
                        <div class="message-content">
                            您好！我是AI助手，有什么可以帮您的吗？
                        </div>
                    </div>
                </div>
                <div class="chat-input-area">
                    <form id="chatForm">
                        <input type="text" name="message" id="messageInput"
                               placeholder="请输入您的问题..." autocomplete="off">
                        <button type="submit">发送</button>
                    </form>
                </div>
            </div>
        </div>
        <script>
        document.getElementById('chatForm').addEventListener('submit', function(e) {
            e.preventDefault();
            var message = document.getElementById('messageInput').value;
            if (!message.trim()) return;

            addMessage('user', message);
            document.getElementById('messageInput').value = '';

            fetch('${pageContext.request.contextPath}/ai?action=send', {
                method: 'POST',
                body: new URLSearchParams({message: message, session_id: '${sessionId}'})
            })
            .then(res => res.json())
            .then(data => {
                if (data.success) {
                    addMessage('assistant', data.response);
                }
            });
        });

        function addMessage(role, content) {
            var messagesDiv = document.getElementById('messages');
            var msgDiv = document.createElement('div');
            msgDiv.className = 'message ' + role;
            msgDiv.innerHTML = '<div class="message-content">' + content + '</div>';
            messagesDiv.appendChild(msgDiv);
            messagesDiv.scrollTop = messagesDiv.scrollHeight;
        }
        </script>
    </body>
    </html>

8.1 技术风险
--------------------------------------------------------------------------------
- AI服务可用性：依赖第三方AI服务，需考虑服务中断时的降级策略
- 响应延迟：AI服务响应时间可能较长，建议添加加载动画和超时处理
- API调用成本：频繁调用AI服务可能产生较高费用，需实现缓存和限流
- 数据安全：用户数据通过AI服务处理，需确保数据加密传输

8.2 安全考虑
--------------------------------------------------------------------------------
- AI服务API Key安全存储（避免硬编码）
- 用户输入内容过滤（防止注入攻击）
- AI输出内容审核（防止敏感信息泄露）
- 会话隔离（用户只能查看自己的对话记录）
- 敏感操作日志记录

8.3 性能优化建议
--------------------------------------------------------------------------------
- 对话历史分页加载，避免一次性加载过多数据
- AI响应结果缓存，相同问题直接返回缓存结果
- 知识库索引优化，加速相关内容检索
- 异步处理AI请求，避免阻塞主线程

8.4 成本预估
--------------------------------------------------------------------------------
- LLM API费用：按调用次数计费，根据用户量预估月度费用
- 服务器资源：如使用本地模型，需考虑GPU服务器成本
- 维护成本：知识库更新、模型调优需要持续投入
